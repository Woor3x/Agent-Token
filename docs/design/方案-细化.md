# Agent 身份与权限系统 — 完整细化方案 v2

## 0. 核心设计原则

| 原则 | 含义 |
|---|---|
| **全动态授权** | 不存在长期 authorization token。每次 A2A 调用实时签发一次性短 TTL token |
| **身份 vs 授权分离** | 身份 = 长期私钥/证书 (signing key)；授权 = 每次动态签 |
| **一次性 token (one-time use)** | OPA 验证通过即销毁 (`jti` 标 used)，防重放，不可复用 |
| **白名单唯一** | Capability/委托/action/resource 全白名单，无黑名单 (黑名单漏掉即风险) |
| **职责分离 (SoD)** | 编排 Agent 无执行能力，执行 Agent 无编排能力，交集 = ∅ |
| **单执行者原则** | 每个 action 策略层绑定唯一 agent_id，防父绕过子直接执行 |
| **Fail-closed** | 任何异常 (OPA 超时/策略缺失/意图解析失败) 默认 deny |
| **可观测优先** | 每次决策 (allow/deny/revoke) 写审计，trace_id 串联全链 |

### 组件职责

| 组件 | 职责 | 无状态? |
|---|---|---|
| **IdP** | 颁发/交换/撤销 token；JWKS；验 Agent client assertion；计算动态交集 scope | 是 (状态在 KMS/Redis) |
| **Gateway (PEP)** | 所有 A2A 调用唯一入口；AuthN + Intent + AuthZ + 一次性销毁 + 审计 | 是 (状态在 Redis) |
| **OPA (PDP)** | 策略决策点；Rego 策略 + 策略数据 (agents/users/executor_map) | 是 |
| **Redis** | `jti:used` (一次性)、`dpop:nonce` (防重放)、`revoked:*` (黑名单)、rate limit buckets | 有状态 |
| **Audit Sink** | SQLite WAL 存审计事件；可选 OTel 导出 | 有状态 |
| **Anomaly Detector** | 订阅审计流，规则命中 → 调 IdP 撤销 | 无状态 |
| **Orchestrator Agent** | 自然语言 → DAG 规划 → 每子任务独立 Token Exchange → 调度 | 无状态 (LLM) |
| **Executor Agent** | 执行具体动作 (飞书 API / Web)；持有唯一底层凭证 | 按 Agent 设计 |

## 2. Token 模型 (全动态 + 一次性)

### 2.1 身份凭证 vs 授权 Token

| 类型 | 形式 | 用途 | 持有期 |
|---|---|---|---|
| **Agent 私钥** | RSA/Ed25519 密钥对 | 证明"我是 Agent X" | 长期 (注册时 KMS 生成) |
| **Client Assertion (JWT-SVID 风格)** | 短 TTL JWT (≤ 60s)，私钥签名 | 调 IdP 换授权 token 的凭证 | 每次调用现签 |
| **User Token** | OIDC ID + Access token | 证明用户身份与权限 | 用户会话期 |
| **Delegated Token (一次性)** | 短 TTL JWT (≤ 120s)，IdP RS256 签 | **唯一**授权介质，调 Gateway 用 | 单次调用，验完即销毁 |

**没有长期 authorization token**。授权能力体现在"能签 client assertion"这件事上，assertion 本身只用于换 delegated token。

### 2.2 Client Assertion (RFC 7521/7523 风格)

```json
{
  "iss": "agent:doc_assistant",          // 自签 (用自己私钥)
  "sub": "agent:doc_assistant",
  "aud": "https://idp.local/token",
  "iat": 1714000000,
  "exp": 1714000060,                      // ≤ 60s
  "jti": "a-uuid"                         // 防重放
}
```

IdP 用 KMS 中对应 Agent 公钥验签。assertion `jti` 在 Redis 标 used (TTL 120s) 防重放。

### 2.3 Delegated Token (一次性授权介质)

```json
{
  "iss": "https://idp.local",
  "sub": "user:alice",                     // 最终受益人
  "act": {                                 // 委托链 (RFC 8693)
    "sub": "agent:doc_assistant",
    "act": null                            // 顶层编排者无上层
  },
  "aud": "agent:data_agent",               // 绑定 callee
  "scope": ["feishu.bitable.read:app_token:bascn.../table:tbl_q1"],
  "purpose": "generate_weekly_report",     // 单次意图摘要
  "plan_id": "plan_01HXYZ",                // DAG 关联
  "task_id": "t1",                         // DAG 节点
  "trace_id": "01HXYZ...",
  "parent_span": "01HXYZ-parent",
  "iat": 1714000000,
  "nbf": 1714000000,
  "exp": 1714000120,                       // ≤ 120s
  "jti": "tok-uuid-v4",
  "cnf": { "jkt": "<caller-DPoP-thumbprint>" },
  "one_time": true                         // 显式标记，Gateway 验完即销毁
}
```

关键:
- `aud` 绑定单一 callee，跨目标调用会被拒
- `scope` 是 IdP 动态算出来的 `user.permissions ∩ orchestrator.caps ∩ requested`
- `one_time=true` + `jti` → OPA allow 后 Redis `SETNX jti:used:<jti>` TTL=exp，重放即 deny

### 2.4 DPoP Sender-Constrained

每次调 Gateway 附 `DPoP` header (caller 用自己私钥签):
```json
{ "htu": "https://gateway/a2a/invoke", "htm": "POST", "iat": ..., "jti": "dpop-uuid", "nonce": "<server-issued>" }
```

Gateway 校验: 签名匹 `cnf.jkt` + `htm/htu` 匹 + `iat` ±60s + `jti` Redis SETNX 防重放 + `nonce` 若 Gateway 先前下发。

## 3. Capability Schema (纯白名单)

### 3.1 Agent Capability 声明

```yaml
# capabilities/data_agent.yaml
agent_id: data_agent
display_name: 企业数据 Agent
role: executor                              # orchestrator | executor
public_key_jwk: { "kty": "RSA", ... }      # IdP 验 client assertion 用

# 静态能力白名单 (能力上限)
capabilities:
  - action: feishu.bitable.read
    resource_pattern: "app_token:*/table:*"
    constraints:
      max_rows_per_call: 1000
      max_calls_per_minute: 60
  - action: feishu.contact.read
    resource_pattern: "department:*"
  - action: feishu.calendar.read
    resource_pattern: "calendar:*"

# 委托接受白名单 (谁可调我)
delegation:
  accept_from:                              # 白名单 only
    - doc_assistant
  max_depth: 3                              # 委托链最大深度

# 底层凭证绑定 (只有此 agent 可拿到)
underlying_credentials:
  - feishu_tenant_token                     # Vault path
```

### 3.2 Orchestrator Agent 声明

```yaml
# capabilities/doc_assistant.yaml
agent_id: doc_assistant
role: orchestrator
public_key_jwk: { ... }

capabilities:
  - action: feishu.doc.write
    resource_pattern: "doc_token:*"
  - action: a2a.invoke
    resource_pattern: "agent:data_agent|agent:web_agent"   # 白名单可调的 executor

delegation:
  accept_from:
    - user                                  # 只接受用户直接委托
  max_depth: 1                              # 本身不可再转授

underlying_credentials:
  - feishu_user_token                       # 写文档代表用户
```

### 3.3 能力正交化 (SoD 静态校验)

CI 脚本:
```python
def test_capability_orthogonality():
    orchestrators = load_agents(role="orchestrator")
    executors    = load_agents(role="executor")
    for o in orchestrators:
        for e in executors:
            o_resource_caps = {c.action for c in o.capabilities if not c.action.startswith("a2a.")}
            e_resource_caps = {c.action for c in e.capabilities}
            overlap = o_resource_caps & e_resource_caps
            assert not overlap, f"SoD violated: {o.agent_id} ∩ {e.agent_id} = {overlap}"
```

### 3.4 User 权限声明

```yaml
# users/alice.yaml
user_id: alice
permissions:
  - action: feishu.bitable.read
```

## 4. 任务规划与多目标授权 (DAG)

### 4.1 规划阶段

Orchestrator 收自然语言 prompt → LLM planner 输出任务 DAG:
```json
{
  "plan_id": "plan_01HXYZ",
  "user": "alice",
  "raw_prompt": "查 Q1 销售表并搜竞品财报写周报",
  "tasks": [
    { "id": "t1", "agent": "data_agent", "action": "feishu.bitable.read", "resource": "app_token:bascn_alice/table:tbl_q1", "params": {...}, "deps": [] },
    { "id": "t2", "agent": "web_agent",  "action": "web.search",          "resource": "*",                                 "params": {"query": "竞品 Q1 财报"}, "deps": [] },
    { "id": "t3", "agent": "doc_assistant", "action": "feishu.doc.write", "resource": "doc_token:weekly",                  "params": {...}, "deps": ["t1","t2"] }
  ]
}
```

### 4.2 DAG 预审 (一次批量 AuthZ)

Orchestrator 签 client assertion + 用户 token 调:
```http
POST /idp/plan/validate
```

IdP 对每节点调 OPA 预检 (不颁 token)，返回每节点 allow/deny + 失败原因。任一 deny → 整 plan 拒执。

### 4.3 执行阶段 (每子任务独立换 token)

无依赖的可并行 (fan-out ≤ 5)：

```
t1: orchestrator 签 assertion_t1 → IdP /token/exchange(task_id=t1) → token1 (aud:data_agent, scope:bitable.read:tbl_q1, one_time)
                                   → Gateway 调 DataAgent → 结果 r1

t2: orchestrator 签 assertion_t2 → IdP /token/exchange(task_id=t2) → token2 (aud:web_agent, scope:web.search:*, one_time)
                                   → Gateway 调 WebAgent → 结果 r2

t3 (依赖 t1,t2 完成):
    orchestrator 签 assertion_t3 → IdP /token/exchange(task_id=t3) → token3 (aud:doc_assistant 自调, scope:doc.write:weekly)
    → 写文档 (r1, r2 作为参数，非 scope)
```

每张 token:
- `aud` 单一
- `scope` 单一
- `one_time=true`，验完销毁
- TTL ≤ 120s

### 4.4 数据不驱动权限膨胀

t1 返回的 bitable 数据仅作为 t3 的**输入参数**，不进入 t3 的 scope。防止"LLM 读到数据里的 id 后伪造新子任务"。

### 4.5 用户确认 (高风险)

DAG 含写/删/跨租户/大批量 → Gateway 暂停 → 推 Web UI 通知用户 → 用户确认后 IdP 才签发对应 token。

## 5. IdP 详细设计

### 5.1 职责与边界

| 归 IdP 管 | 不归 IdP 管 |
|---|---|
| Agent 注册、密钥对生成/轮换 | Agent 业务逻辑 |
| User OIDC 登录、Access Token 颁发 | User 密码校验 (委托下游 IdP 或本地 bcrypt) |
| Client Assertion 验证 (RFC 7523) | 调用真实资源 |
| **Token Exchange** (RFC 8693) — 动态交集计算 + 一次性 delegated token 颁发 | 资源访问 (归 Gateway + Executor Agent) |
| DAG 预审 (调 OPA) | OPA 策略维护 (独立仓库管理) |
| 撤销 (jti/sub/agent/trace/plan/chain) | 撤销决策规则 (由管理员/anomaly detector 触发) |
| JWKS 分发 | Token 使用状态 (一次性由 Gateway 负责) |
| 所有颁发/拒发/撤销事件审计 | 业务行为审计 (Gateway 负责) |

IdP = 身份 + 授权决定源。**不持有**业务资源凭证。飞书 tenant_token 归 DataAgent，IdP 无权接触。

### 5.3 存储模型

| 存储 | 内容 | 访问模式 |
|---|---|---|
| **KMS (文件 + age 加密 / Vault)** | Agent 私钥、IdP 签名密钥 (current + previous for rotation) | 只 IdP 进程读；轮换由 admin 触发 |
| **Redis** | `assertion:jti:<j>` (TTL 120s 防 assertion 重放), `revoked:jtis` / `revoked:subs` / `revoked:agents` / `revoked:traces` / `revoked:plans` / `revoked:chains` (SET), `rate:idp:<agent>:<path>` (token bucket) | 毫秒级 |
| **SQLite (WAL)** | `agents` (注册信息), `users` (permissions), `jwks_rotation` (历史), `audit` (颁发/拒发/撤销) | 写多读少 |

### 5.5 密钥管理 (KMS)

#### 5.5.1 三类密钥

| 密钥 | 用途 | 算法 | 轮换周期 |
|---|---|---|---|
| **IdP 签名密钥** | 签 delegated token / user access token | RS256 (RSA-2048) | 90 天 |
| **Agent 私钥** | Agent 签 client assertion | RS256 或 EdDSA | 180 天 |
| **IdP TLS 证书** | mTLS 双向认证 | 自签 CA | 365 天 |

#### 5.5.2 密钥轮换 (IdP 签名密钥)

```
t=0     : key_v1 active, JWKS=[v1]
t=rotate: 生成 key_v2
          JWKS=[v1, v2]  ← 双密钥期，所有 verifier 缓存刷新 (10min)
t=+15min: 新 token 签 v2; 旧 token 仍可验
t=+TTL_max (120s): v1 再无活跃 token → JWKS=[v2]
          archive v1
```

同理 Agent 私钥轮换:
- 新密钥上线前，Agent YAML 的 `public_key_jwk` 改成数组支持双公钥
- IdP 验 assertion 时按 `kid` 匹配
- 宽限期后老 key 撤废

#### 5.5.3 密钥存储

演示环境: `services/idp/kms/keys/` 目录，每文件用 `age` 加密，key 来自环境变量 `IDP_KMS_PASSPHRASE`。

生产选项: HashiCorp Vault Transit Engine；IdP 不直接持私钥，签名调 Vault。

### 5.7 Client Assertion 验证 (RFC 7523)

```python
# services/idp/client_assertion.py
async def verify_client_assertion(assertion_jwt: str) -> AgentIdentity:
    # 1. 解 header 取 kid
    header = jwt.get_unverified_header(assertion_jwt)
    kid = header["kid"]

    # 2. 加载 Agent 公钥
    agent_row = await db.get_agent_by_kid(kid)
    if not agent_row:
        raise Invalid("unknown_kid")
    if agent_row.status != "active":
        raise Invalid("agent_disabled")

    # 3. 验签 + 必要 claim
    claims = jwt.decode(
        assertion_jwt,
        key=agent_row.public_jwk,
        algorithms=[agent_row.alg],
        audience="https://idp.local/token",   # 必须是 IdP token 端点
        issuer=agent_row.agent_id,              # iss 必须 = agent_id
        leeway=30,
    )

    # 4. 时间窗 exp-iat ≤ 60s
    if claims["exp"] - claims["iat"] > 60:
        raise Invalid("assertion_too_long")

    # 5. sub == iss (self-issued)
    if claims["sub"] != claims["iss"]:
        raise Invalid("sub_iss_mismatch")

    # 6. 防重放 (Redis SETNX)
    if await redis.set(f"assertion:jti:{claims['jti']}", 1, nx=True, ex=120) is None:
        raise Invalid("assertion_replay")

    # 7. 撤销检查
    if await redis.sismember("revoked:agents", agent_row.agent_id):
        raise Invalid("agent_revoked")

    return AgentIdentity(agent_id=agent_row.agent_id, kid=kid, jti=claims["jti"])
```

#### 5.8.2 算法 (完整)

```python
# services/idp/token_exchange.py
async def token_exchange(req: TokenExchangeRequest, dpop_header: str) -> dict:
    # === Phase 1: 身份验证 ===
    orchestrator = await verify_client_assertion(req.client_assertion)
    user = await verify_subject_token(req.subject_token)   # 验用户 token 签名 + exp

    # === Phase 2: DPoP 绑定 ===
    dpop_claims = verify_dpop_proof(
        dpop_header,
        htu="https://idp.local/token/exchange",
        htm="POST",
        max_skew=60,
    )
    dpop_jkt = dpop_claims["jkt"]  # 用作 cnf.jkt

    # === Phase 3: 意图解析 ===
    action, resource = parse_scope(req.scope)              # "action:resource"
    if action not in ACTION_ENUM:
        raise Forbidden("unknown_action")
    if not RESOURCE_REGEX[action].match(resource):
        raise Forbidden("resource_format")

    # === Phase 4: 委托合法性 ===
    orch_caps = await load_capabilities(orchestrator.agent_id)
    # orchestrator 必须白名单可 a2a.invoke 目标
    if not whitelist_match(orch_caps, "a2a.invoke", f"agent:{req.target_agent}"):
        raise Forbidden("delegation_not_allowed")

    # callee 必须白名单接受 orchestrator
    callee_meta = await load_agent(req.target_agent)
    if orchestrator.agent_id not in callee_meta.delegation.accept_from:
        raise Forbidden("callee_rejects_caller")

    # === Phase 5: 单执行者校验 ===
    if EXECUTOR_MAP[action] != req.target_agent:
        raise Forbidden("executor_mismatch")

    # === Phase 6: 动态交集 ===
    callee_caps = await load_capabilities(req.target_agent)
    user_perms  = await load_permissions(user.sub)
    effective   = intersect(callee_caps, user_perms, [(action, resource)])
    if not effective:
        raise Forbidden("empty_effective_scope")

    # === Phase 7: 上下文约束 ===
    effective = apply_context(effective, {
        "time": now(),
        "user": user.sub,
        "trace_id": req.trace_id,
        "client_ip": req.client_ip,
        "plan_id": req.plan_id,
    })
    if not effective:
        raise Forbidden("context_denied")

    # === Phase 8: 配额 ===
    if not await rate_limit_ok(orchestrator.agent_id, action):
        raise RateLimited()

    # === Phase 9: 签发一次性 delegated token ===
    now_s = int(time.time())
    claims = {
        "iss": "https://idp.local",
        "sub": user.sub,
        "act": { "sub": orchestrator.agent_id, "act": None },
        "aud": f"agent:{req.target_agent}",
        "scope": effective,
        "purpose": req.purpose,
        "plan_id": req.plan_id,
        "task_id": req.task_id,
        "trace_id": req.trace_id,
        "parent_span": req.parent_span,
        "iat": now_s, "nbf": now_s, "exp": now_s + 120,
        "jti": str(uuid.uuid4()),
        "cnf": { "jkt": dpop_jkt },
        "one_time": True,
        "policy_version": POLICY_VERSION,
    }
    signing_key = kms.get_active_signing_key()
    token = jwt.encode(claims, signing_key.private_pem, algorithm="RS256",
                       headers={"kid": signing_key.kid})

    # === Phase 10: 审计 ===
    await audit.write({
        "event_type": "token.issue",
        "jti": claims["jti"], "sub": user.sub,
        "act": orchestrator.agent_id, "aud": claims["aud"],
        "scope": effective, "purpose": req.purpose,
        "plan_id": req.plan_id, "task_id": req.task_id,
        "trace_id": req.trace_id, "policy_version": POLICY_VERSION,
        "dpop_jkt": dpop_jkt, "ip": req.client_ip,
        "ts": now_s,
    })

    return {
        "access_token": token,
        "issued_token_type": "urn:ietf:params:oauth:token-type:jwt",
        "token_type": "DPoP",
        "expires_in": 120,
        "jti": claims["jti"],
    }
```

#### 5.8.3 交集算法

```python
def intersect(callee_caps: list, user_perms: list, requested: list[tuple]) -> list[str]:
    result = []
    for (action, resource) in requested:
        callee_ok = any(cap_match(c, action, resource) for c in callee_caps)
        user_ok   = any(cap_match(p, action, resource) for p in user_perms)
        if callee_ok and user_ok:
            # 此外应用 constraints (max_rows 等)
            result.append(f"{action}:{resource}")
    return result

def cap_match(cap, action, resource):
    if cap.action != action:
        return False
    return glob.fnmatch(resource, cap.resource_pattern)
```

#### 5.8.4 上下文约束 `apply_context`

```python
def apply_context(scope, ctx):
    # 非工作时间拒发写类 scope
    if is_write_action_any(scope) and ctx["time"].hour < 6:
        return []
    # 用户短时高频 → 降级
    if recent_count(ctx["user"], window=60) > 200:
        return []
    # IP 白名单
    if ctx["client_ip"] not in ALLOWED_SOURCE_NETS:
        return []
    return scope
```

```python
async def revoke(req):
    check_admin_or_service_auth(req)
    key_map = {
        "jti":   "revoked:jtis",
        "sub":   "revoked:subs",
        "agent": "revoked:agents",
        "trace": "revoked:traces",
        "plan":  "revoked:plans",
        "chain": "revoked:chains",
    }
    await redis.sadd(key_map[req.type], req.value)
    await redis.expire_member(key_map[req.type], req.value, req.ttl_sec)  # Redis 7 SETEX per-member
    await redis.publish("revoke", json.dumps({"type": req.type, "value": req.value}))
    await audit.write({ "event_type": "token.revoke", **req.__dict__ })
    return { "revoked": True, "effective_at": now() }
```

## 6. OPA 策略 (PDP)

### 6.1 核心规则

```rego
package agent.authz

import future.keywords.if
import future.keywords.in

default allow := false
default reasons := []

# 1. Token 形式 (IdP 已验签，OPA 验语义)
token_valid if {
    input.token.iss == "https://idp.local"
    input.token.exp > time.now_ns() / 1e9
    input.token.nbf <= time.now_ns() / 1e9
    input.token.one_time == true
}

# 2. 一次性: 必须未 used (由 Gateway 在 OPA 调用前查 Redis，这里只确认 token 声明)
one_time_declared if {
    input.token.one_time == true
    input.token.jti != ""
}

# 3. Audience 匹配: token 绑定的 aud 必须等于当前 callee
audience_match if {
    input.token.aud == sprintf("agent:%s", [input.target_agent])
}

# 4. Scope 覆盖意图
scope_covers if {
    required := sprintf("%s:%s", [input.intent.action, input.intent.resource])
    some s in input.token.scope
    glob_match(s, required)
}

# 5. 单执行者: action 的唯一允许执行者
executor_valid if {
    data.executor_map[input.intent.action] == input.target_agent
}

# 6. 委托白名单: callee 白名单接受 caller
delegation_accepted if {
    caller := input.token.act.sub
    data.agents[input.target_agent].delegation.accept_from[_] == caller
}

# 7. 委托链深度
delegation_depth_ok if {
    count(actor_chain(input.token.act)) <= data.agents[input.target_agent].delegation.max_depth
}

actor_chain(act) := chain if {
    act == null
    chain := []
} else := chain if {
    act != null
    sub_chain := actor_chain(act.act)
    chain := array.concat([act.sub], sub_chain)
}

# 8. 未撤销 (Gateway 预查 Redis，OPA 作为二次确认)
not_revoked if {
    not data.revoked.jtis[input.token.jti]
    not data.revoked.subs[input.token.sub]
    not data.revoked.traces[input.token.trace_id]
}

# 9. DPoP 绑定 (Gateway 已验签，OPA 确认 cnf 存在)
dpop_bound if {
    input.token.cnf.jkt != ""
}

# 10. 上下文约束 (时间/配额)
context_ok if {
    not rate_limit_exceeded
    not out_of_hours_write
}

rate_limit_exceeded if {
    input.context.recent_calls > data.agents[input.target_agent].capabilities[_].constraints.max_calls_per_minute
}

out_of_hours_write if {
    startswith(input.intent.action, "feishu.doc.write")
    hour := time.clock(time.now_ns())[0]
    hour < 6
}

# 综合
allow if {
    token_valid
    one_time_declared
    audience_match
    scope_covers
    executor_valid
    delegation_accepted
    delegation_depth_ok
    not_revoked
    dpop_bound
    context_ok
}

# 原因 (deny 时返回)
reasons contains "token_expired"          if not token_valid
reasons contains "audience_mismatch"      if not audience_match
reasons contains "scope_exceeded"         if not scope_covers
reasons contains "executor_mismatch"      if not executor_valid
reasons contains "delegation_rejected"    if not delegation_accepted
reasons contains "depth_exceeded"         if not delegation_depth_ok
reasons contains "revoked"                if not not_revoked
reasons contains "context_denied"         if not context_ok
```

### 6.3 一次性销毁 (Gateway 层)

OPA 返回 allow 后，Gateway **立即**:
```python
if not await redis.set(f"jti:used:{token.jti}", 1, nx=True, ex=token.exp - now()):
    # 已 used → 重放攻击
    raise AuthnError("TOKEN_REPLAYED")
```

## 7. Gateway 详细设计 (PEP)

### 7.1 中间件链

```
Request ─▶ 1. Transport Adapter (HTTP/gRPC/MCP)
       ─▶ 2. Body Parser + Size Limit (256KB)
       ─▶ 3. AuthN
              ├─ JWT 验签 (JWKS cache 10min，LRU)
              ├─ exp/nbf/aud/iss 校验
              ├─ 撤销查: SISMEMBER revoked:jtis / revoked:subs / revoked:traces
              └─ DPoP 校验 (htu/htm/iat±60s/jti SETNX)
       ─▶ 4. Rate Limit (token bucket per agent_id+action, Redis)
       ─▶ 5. Intent Parser (JSON Schema 严格校验 + action enum + resource regex)
       ─▶ 6. Delegation Chain Verifier (递归解析 act + max_depth)
       ─▶ 7. AuthZ Client → OPA (HTTP, 5ms timeout, fail-closed)
       ─▶ 8. One-Shot Consume (SETNX jti:used:<jti> TTL=exp) ← token 销毁
       ─▶ 9. Trace Injector (W3C traceparent + baggage)
       ─▶ 10. Router (registry.yaml: agent_id → upstream URL)
       ─▶ 11. Circuit Breaker (per upstream)
       ─▶ 12. Upstream Call (httpx AsyncClient, mTLS)
       ─▶ 13. Response Sanitizer (剥敏感 header / 脱敏)
       ─▶ 14. Audit Writer (asyncio.Queue 批量写 SQLite)
       ─▶ Response
```

|---|---|---|

## 10. Agent 行为不可预测性防御

### 10.1 风险矩阵

| 风险 | 缓解 |
|---|---|
| Prompt Injection / 目标漂移 | 意图 schema 白名单 + purpose 绑定 + 原始 prompt 审计 |
| 幻觉调用 | action enum + resource regex + OPA executor_map 拒未知 |
| 越权试探 | 异常检测: 连续 N deny → 撤销 sub |
| 死循环 | max_depth + trace 环检测 |
| 数据外泄 | aud 绑定 + 出站白名单 + 响应脱敏 |
| 长期滥用 | 无长期 authorization token (根治) |

### 10.2 Token 一次性 + 实时撤销

**一次性**: `jti:used` Redis SETNX，验完即毁 → 即使泄漏最多用一次。

**实时撤销** (补充):

```
┌──────────────┐     ┌────────────────┐     ┌──────────────────┐
│ Admin UI / CLI│    │ IdP            │     │ Redis            │
│ Anomaly       │───▶│ POST /revoke   │────▶│ revoked:jtis     │
│ Detector      │    │                │     │ revoked:subs     │
│ User logout   │    └───────┬────────┘     │ revoked:traces   │
└──────────────┘            │               │ revoked:chains   │
                            ▼               └─────────┬────────┘
                   ┌────────────────┐                 │
                   │ Redis Pub/Sub  │──broadcast─────▶│ Gateway 本地 bloom filter 热更
                   └────────────────┘
```

### 10.3 撤销粒度

| 粒度 | 键 | 用途 |
|---|---|---|
| `jti` | `revoked:jtis` | 单 token |
| `sub` | `revoked:subs` | 用户整体封禁 |
| `agent_id` | `revoked:agents` | Agent 整体封禁 |
| `trace_id` | `revoked:traces` | 任务链紧急止血 |
| `plan_id` | `revoked:plans` | DAG 整体终止 |

## 11. Prompt Injection 防御

1. **System prompt 固定** + user prompt 以 `<user_input>` 定界
2. **LLM 输出强约束** — tool-calling 模式 + JSON Schema 强制，失败 INTENT_INVALID
3. **action enum + resource regex** — LLM 即使越狱也无法产生非白名单 action
4. **数据不驱动权限** — 工具返回数据进入 LLM 上下文，但新子任务仍需过 plan/validate
5. **原始 prompt 审计** — 评估员可复核

## 15. 演示场景

### A. 正常委托
1. Alice 登录 → user token
2. "生成 Q1 销售周报" → DocAssistant planner 产 DAG (t1: bitable.read, t2: doc.write)
3. DAG validate 通过
4. t1: sign assertion → IdP 换 token1 (aud:data_agent, one_time) → Gateway → DataAgent → 飞书 API → 数据返回
5. t2: sign assertion → IdP 换 token3 (doc_assistant 自调) → Gateway → 写文档
6. Web UI 可见 DAG 全绿 + 2 条 audit allow + 2 条 token issue/consume

### B. 越权拦截
1. WebAgent 被诱导尝试 `a2a.invoke:data_agent`
2. WebAgent 签 assertion 调 IdP `/token/exchange(target=data_agent)`
3. IdP 查 WebAgent capabilities 无 `a2a.invoke:data_agent` → 拒发
4. 或: WebAgent 直调 Gateway → OPA `executor_mismatch` (action=bitable.read 但 target=web_agent 不是 data_agent)
5. 返回 `403 AUTHZ_EXECUTOR_MISMATCH`，审计 deny

### C. Token 重放
1. 攻击者截获已使用的 delegated token
2. 重放 → Gateway SETNX 失败 → `401 TOKEN_REPLAYED`
3. 审计记录

### D. DPoP 盗用
1. 攻击者仅拿到 token，无私钥
2. 自制 DPoP 签名 → Gateway 验签失败 `cnf.jkt` 不匹 → `401 AUTHN_DPOP_INVALID`

### E. 异常检测自动撤销
1. 注入 → WebAgent 连续 5 次尝试换 data_agent token 全拒
2. anomaly detector `consecutive_deny` 触发 → `/revoke {type:"agent",value:"web_agent"}`
3. 后续 WebAgent 任何请求 → 401 AUTHN_REVOKED
4. Web UI 撤销面板可见

### F. 多目标 DAG 并行
1. "查 Q1 销售 + 搜竞品财报写周报"
2. DAG: t1 ∥ t2 → t3
3. t1、t2 并行签 assertion 各换独立 token，并行调
4. 完成后 t3 聚合
5. DAG 可视化展示并行 + 聚合节点

## 19. 主要变更 (vs v1)

1. **取消长期 Agent 身份 Token**，改用 client assertion (RFC 7523) 每次现签
2. **只保留动态 delegated token**，强制一次性 (`one_time=true` + SETNX)
3. **Capability 纯白名单** — 删除 `reject_from`
4. **引入 executor_map** — 单执行者绑定，防父绕过子
5. **新增 DAG 规划与多目标授权** — 每子任务独立换 token
6. **OPA 增加 one_time/executor/depth 规则**
7. **Gateway 新增 One-Shot Consume 中间件**
8. **IdP 新增 `/plan/validate` 端点**
9. **数据不驱动权限膨胀** 原则明确
