# 阶段小结（M4 Agents 编码阶段）

## 一、核心产出

讨论分析完成了整体框架结构的方案搭建，深入研究讨论了动态授权的形式和 OPA 内部功能，由原来的"动+静结合"演进到"动态一次性授权"，确定了最终的设计开发方案以及分工调配，然后完成了 agents 相关的代码编写。

最满意的一段是 `agents/common/server.py` + `agents/common/auth.py` 这套公共骨架 —— 三个 agent（doc_assistant / data_agent / web_agent）共用一个 FastAPI server 模板，同一份 DPoP 验证 + 上游 token 透传逻辑，每个 agent 只写自己的 `handler.py` 业务代码。doc_assistant 用 LangGraph 串了 4 个节点（planner → plan_validate → dispatcher → synthesizer + doc_writer），fan-out 调 data/web/feishu 时全自动走 SDK 重新 token-exchange，零信任那条"每一跳重新授权"算是第一次落到代码里跑出来。

## 三、过程复盘与沉淀

### 1. 这周主要搞定的环节

把题目里"零信任 + Agent + token"这几个词的语义拆开。零信任意味着每一跳都要重新授权，不能信任前一跳。

AI 提议直接引用 RFC 8693 token-exchange 做核心，actor token 表示委托链；DPoP（RFC 9449）做持有者绑定，避免 token 被偷了能转手。

具体到 agents 代码这一块：
- **公共骨架抽到 `agents/common/`**：`server.py` 统一 FastAPI 启动 + `/invoke` 路由；`auth.py` 校验上游传来的 X-Subject-Token + DPoP；`bootstrap_register.py` 启动时自注册到 IdP；`capability.py` 解析 capability.yaml；`llm/` 封装 Claude / 火山方舟 / OpenAI 三家 provider
- **doc_assistant 用 LangGraph**：`graph.py` 定状态机，`nodes/` 下每个节点单独一个 .py，方便 AI 按节点拆开生成
- **data_agent / web_agent 是叶子节点**：`handler.py` 直接调下游（feishu-mock / SerpAPI mock），不需要再 fan-out

跟 AI 分工：
- 让 AI 把 RFC 8693 的请求/响应字段直接套到业务场景里输出对照表
- 让 AI 检查方案里"撤销机制"那段有没有遗漏（它指出我没考虑 trace_id 维度撤销，补上了）
- LangGraph 节点 prompt 让 AI 起初稿，自己改边界（json schema 校验、错误回退）
- handler.py 里调下游 SDK 的样板代码 AI 直接生成

### 2. 卡住的地方

一开始想"一个 token 一个 scope"做到极致 —— 每次调用都重新签一个一次性 token。后来发现编排 agent 同时 fan-out 三个执行 agent 时，要签三个不同 aud 的 token，IdP 一次返回不了，串行又慢。

破局：分两种情况。用户 → 编排 agent 这一跳签普通 delegated token（短 TTL，多次复用）；编排 agent → 执行 agent 做成一次性 token（cnf.jkt 绑 DPoP，jti 用一次进 Redis revoked）。

落到 agents 代码里就是 doc_assistant 的 `nodes/dispatcher.py` —— 它对每个 fan-out 目标单独调 SDK，每次 SDK 内部都重新走一遍 `/token/exchange` 拿 one-shot token，三个并发 `asyncio.gather`，串行变并行。

### 3. 复用沉淀

- 给 AI 一段需求描述 + 一个示例 API 表行，让它按表格行格式输出新接口，比让它写自由 markdown 整齐多了
- agent 三件套模板（`main.py` 极简 / `handler.py` 业务 / `capability.yaml` 声明能力），下次再加新 agent 直接 copy paste，半小时就能起一个
- LangGraph 节点单文件单职责的写法 —— 一个节点一个 .py，节点间只通过 state dict 传数据，AI 改某个节点不会动到别的

## 四、本周随手记

- 公共骨架抽得越早越省事。data_agent 和 web_agent 写完之后才把共用部分抽到 `agents/common/`，回头改 doc_assistant 时多花了一晚上重构
- LangGraph 的 state 用 TypedDict 比 dict 好 —— 改字段名时 AI 自己能跟着改完整链路，dict 改了字段就到处漏
- agent 自注册 `bootstrap_register.py` 用 volume 缓存私钥这套，第一次跑没 volume 就直接挂了。后续要做"已注册时静默跳过"的判断
- LLM provider 抽象其实可以再退一步，初期直接写一个 mock provider，开发期完全不调真模型，省 token 又快
