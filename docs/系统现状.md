# 系统现状（M1 + M2 + M4 + M5 联调后）

## 一、已实现

### M1 — IdP（身份提供方，FastAPI）
- **OIDC PKCE 登录**：`/oidc/authorize`、`/oidc/login`、`/oidc/token`、`/oidc/callback`，bcrypt 校验用户密码
- **Agent 注册 / 轮换**：`/agents/register` 接受 capability YAML，IdP 生成 RS256 密钥对，KMS 加密落 SQLite；`/agents/rotate` 轮换私钥
- **JWKS 端点**：`/jwks` 输出公钥集合，下游网关拉取
- **RFC 8693 Token Exchange**：`/token/exchange`，校验 client_assertion (RFC 7521/7523)，按 capability + 用户权限 + 请求三方求交集，签 delegated token
  - 输出 token 含 `sub / act / aud / scope / cnf.jkt / plan_id / task_id / trace_id / one_time / jti`
- **撤销 API**：`/admin/revoke`，4 维（jti / sub / trace / plan）写 Redis Set，Pub/Sub 通知 Gateway
- **审计**：每次发 token / 注册 agent 落 SQLite WAL
- **Rate limit**：每 agent 每 action 60s 100 次（Redis incr）

### M2 — Gateway（PEP，FastAPI）
- **AuthN 中间件**：JWKS 验签 → 4 维撤销查询 → DPoP 验证（htu/htm/iat/jti SETNX 防重放）→ aud 与 X-Target-Agent 比对
- **OPA 策略引擎调用**：10 条规则
  - token_valid / one_time_declared / audience_match / scope_covers / executor_valid / delegation_accepted / delegation_depth_ok / not_revoked / dpop_bound / context_ok
- **One-shot 销毁**：通过授权后立即把 jti SADD 进 `revoked:jtis`
- **路由 + 熔断**：`registry.yaml` 静态注册 3 个 agent，httpx 转发，circuit breaker 失败计数
- **审计**：异步批写 SQLite WAL（`/data/gateway-audit.db`）
- **管理端**：`/admin/revoke`、`/admin/registry`，需 admin token
- **NL parser**：`/nl/parse` 走 Anthropic Claude 把自然语言转 intent（已接入但未在 demo 里用）
- **Plan 端点**：`/plan/*`，多步规划骨架
- **健康 / Metrics**：`/healthz`、`/metrics`（Prometheus）

### M4 — Agents（3 个 FastAPI 服务）
- **doc_assistant**（编排）：LangGraph 节点图，把 prompt → 计划 → fan-out 到 data_agent / web_agent / feishu-mock → 合成 doc
- **data_agent**：调 feishu-mock 拉 bitable / contact / calendar
- **web_agent**：搜索 + 抓页（带 SSRF 黑名单 + robots 检查）
- **公共能力**：所有 agent 启动时自注册到 IdP（`scripts/agent_entrypoint.sh`），`/invoke` 接收上游调用，再用 SDK 起新 token-exchange 调下游

### M5 — Mocks
- **feishu-mock**：bitable / contact / calendar / docx 接口模拟，返回固定 fixture
- **gateway_mock / idp_mock**：早期联调用，已被真 M1/M2 替代，保留供单元测试

### SDK
- `AgentClient.invoke(target, intent, on_behalf_of)`：自动 token-exchange + DPoP 签名 + POST 网关
- 自动多跳 subject_token 透传（`X-Subject-Token` 头）
- 重试包装 `invoke_with_retry`

### 基础设施
- `docker-compose.yml` 一键起 8 服务，volumes 持久化 IdP/KMS/Gateway 审计
- `scripts/e2e_demo.py` 端到端走 OIDC → 编排 → fan-out → 返 doc，约 1.3 s
- `pytest tests/` **66 通过**

## 二、未实现 / 不全的部分

### 功能缺口
1. **M3 独立审计服务**：当前 IdP 和 Gateway 各落各的 SQLite，没有统一的 audit-collector 服务，没法跨服务关联查询
2. **审计查询 CLI / UI**：审计落了但只能 `sqlite3` 直接查，没有 `audit-cli` 工具或 Web 看板
3. **Web UI**：只有 CLI demo（`e2e_demo.py`），没有真正的浏览器界面给用户登录 / 看 agent 列表 / 触发任务
4. **mTLS 双向认证**：Gateway 配置项 `GW_MTLS_ENABLED=false`，证书生成脚本和 CA 没做
5. **Plan 多步规划真实落地**：`gateway/routes/plan.py` 是骨架，doc_assistant 内部用 LangGraph 串了，但 Gateway 层 plan-token（一个 plan_id 派生多 task_id）没真跑
6. **NL parser 没接入主链路**：`/nl/parse` 端点存在，但 demo 还是用结构化 intent
7. **Capability 热更新**：agent capability YAML 改了要重启容器，IdP 没监听文件变化
8. **OPA 数据热加载**：`services/opa/data/*.json` 改了必须 `docker compose restart opa`，没用 bundle / discovery
9. **Token rotation 自动化**：IdP 有 `/agents/rotate` 端点但没定时任务触发
10. **多租户 / 隔离**：当前只有"用户 alice"，没有租户概念

### 测试缺口
- Gateway 模块自带的 `tests/` 目录没有，只有顶层 `tests/`（覆盖 SDK + agent + IdP 端到端）
- 没有性能 / 压力测试（虽然有 `gateway/bench.py` 但没跑）
- 没有混沌测试（OPA 挂、Redis 挂、IdP 挂时的行为只在代码里 `fail-closed`，没实测过）

### 文档缺口
- 没画系统总时序图（OIDC + token-exchange + invoke + fan-out 全流程）
- API 接口清单写过但 M2 加进来后没更新
- 部署文档只有 `docker compose up`，没生产化指南（K8s manifest、TLS 终结、密钥外置）

## 三、已知缺陷

### 安全 / 正确性
1. **Gateway 包名 `token` → `jwttoken` workaround**：因为撞 stdlib `token` 模块改名，上游 M2 仓再拉新代码会冲突。需要把 patch 维护成补丁集，或者向上游 PR
2. **DPoP `verify_dpop` 用 `expected_htu=str(request.url)`**：如果反向代理改写 URL（host / scheme），会校验失败。生产部署需要走 `X-Forwarded-*` 头解析
3. **OPA `audience_match` 改成 bare aud**：M1 的契约决定，但偏离了 M2 上游的 `agent:{id}` 写法。如果未来对接其他网关，aud 命名不统一
4. **executor_map 单值 + a2a.invoke 特判**：`a2a.invoke` 在 Rego 里靠 `trim_prefix(resource, "agent:")` 提目标，意味着 resource 字段格式被硬编码。换种 resource 写法（比如 `agent://data_agent`）就会失效
5. **Bloom filter 撤销查询的假阳性没监控**：`revoke/bloom.py` 用布隆过滤器加速 jti 查询，假阳性率没暴露到 metrics
6. **One-shot consume 与 OPA decision 之间不是原子**：先 OPA → 再 SADD jti 进 revoked。中间崩了 token 没销毁，理论上能复用一次
7. **审计写 SQLite WAL，Gateway 重启可能丢未刷盘的 batch**：`audit_flush_interval_ms=100`，最坏 100ms 数据丢失

### 工程 / 运维
8. **没 healthcheck 的 gateway 容器**：compose 里 gateway 没配 `healthcheck`，依赖它的服务用 `service_started` 凑合
9. **`agent_entrypoint.sh` 注册逻辑非幂等**：agent 已注册再启会失败（IdP 拒重复 register），靠 volume 缓存私钥规避，volume 丢就完蛋
10. **gateway audit DB 单文件**：高并发会成为瓶颈，没分片 / 转发到 Kafka 等
11. **Redis 单点 + 无密码**：`GW_REDIS_PASSWORD` 空，`redis://redis:6379` 没 TLS
12. **CORS 没配**：gateway 加了 `CORSMiddleware` import 但没启用，浏览器场景待补
13. **`__pycache__` 进了 git 工作区**：`.gitignore` 加了但已存在的 pycache 没清

### 可用性
14. **首跑 web_ui 注册依赖 `.demo_keys/` 目录**：删了之后再跑会因 IdP 拒重复 register 而崩
15. **OPA 改 data 后不热加载**：dev 流程要 `docker compose restart opa`，新人容易忘
16. **e2e demo 错误信息不友好**：失败只抛 `AUTHZ_SCOPE_EXCEEDED` + reasons 列表，没有"哪条策略具体怎么不满足"的解释
